## 2025-11-06 -- Initial Mailbox Export

First commit. The entire static bundle was generated by the MCP Agent Mail `share export` pipeline (exporter version `prototype`, manifest schema `0.1.0`) on 2025-11-06.

([`9857721`](https://github.com/Dicklesworthstone/agent-mailbox-viewer-example/commit/98577212eb6c86780dc9ac5705ac88704d05f24a))

This section is organized by capability area rather than file-diff order.

### Mailbox Snapshot

- `mailbox.sqlite3` -- 4.7 MB scrubbed SQLite database containing **1,500 messages** across **66 agents**, exported from the `mcp_agent_mail_ios_app` project scope.
- Scrub preset **standard**: ack flags cleared (1,500), agent links removed (398), file reservations removed (289), recipients cleared (3,242). Agent names are retained; no body redaction or pseudonymization was applied.
- `manifest.json` -- machine-readable export metadata recording database SHA-256, scrub statistics, project scope, hosting auto-detection hints, and SHA-256 SRI hashes for every viewer asset.
- `manifest.sig.json` -- Ed25519 signature over the manifest for end-to-end bundle integrity verification (public key embedded).

### In-Browser SQLite Engine

- Loads `mailbox.sqlite3` directly in the browser via **sql.js v1.10.1** (SQLite compiled to WebAssembly). All queries execute client-side with zero server involvement.
- **OPFS caching** -- after the first network fetch, the database bytes are written to the Origin Private File System. Subsequent visits load from OPFS, validated by SHA-256 against `manifest.json`. The cache can be toggled on or off from the toolbar.
- **Chunked database support** -- large databases can be split into numbered chunks (`formatChunkPath`); the loader fetches and reassembles them transparently.
- **Query plan diagnostics** -- an `explainMode` flag logs `EXPLAIN QUERY PLAN` output for every SQL query to the browser console, aiding performance debugging.

### Viewer UI and Navigation

- **Three view modes**: Split (Gmail-style message list + detail pane), List (message list only), and Threads (thread explorer sidebar + threaded conversation view). Switchable from a segmented control in the toolbar.
- **Responsive mobile layout** -- media-query breakpoint at 768 px, scroll-aware auto-collapse of the filter panel, and a full-screen modal overlay for reading messages on small screens. Body scroll is locked while the mobile message overlay is open.
- **Virtual scrolling** -- custom implementation with estimated-row-height refinement, configurable overscan, and `requestAnimationFrame`-throttled rendering. Handles large inboxes (tested with 1,500 messages) without DOM bloat. Row heights are measured with an exponential moving average to reduce layout jumps after font loading. (The vendored Clusterize.js CSS is retained for potential fallback; its JS is superseded.)
- **Dark mode** -- toggle persisted to `localStorage`; respects `prefers-color-scheme` on first visit. A synchronous inline `<script>` in `<head>` prevents flash of unstyled content (FOUC). Full dark-mode typography rules for prose, code blocks, blockquotes, tables, and list markers.

### Thread Explorer

- Groups messages by `thread_id`; standalone messages get synthetic keys (`msg:<id>`).
- Displays per-thread message count and latest snippet.
- Thread search filters by subject, thread identifier, and snippet text.
- Selecting a thread scrolls its entry into view with smooth scrolling.
- Threads are categorized as `user`, `admin`, or `mixed` based on the administrative status of their constituent messages.

### Search

- **Advanced boolean search** -- supports `AND`, `OR`, `NOT` operators, parenthesized grouping, and double-quoted phrase matching.
- Internally uses a **shunting-yard parser** to build an AST (precedence: NOT > AND > OR), then generates either an FTS5 `MATCH` expression (when the database has an `fts_messages` table) or a `LIKE`-based SQL fallback.
- Input is **debounced at 140 ms** to avoid hammering sql.js on rapid keystrokes.

### Multi-Axis Filtering

- Filter by **project**, **sender**, **recipient** (exact match against comma-separated recipient lists), **importance** level (urgent / high / normal / low with per-level counts), **thread membership** (threaded vs. standalone), and **message kind** (user vs. administrative).
- Administrative messages are auto-detected by regex patterns matching subject lines (`contact request from`, `auto-handshake`) and body text.
- Active-filter indicator (pulsing dot) on the Filters button; one-click "clear all filters" resets every axis.
- Unique filter options are derived dynamically from the loaded dataset.

### Sorting

Five sort modes available from a dropdown: newest first, oldest first, subject (alphabetical), sender (alphabetical), and longest body.

### Bulk Actions

- Select-all / individual message checkbox selection.
- Mark-as-read updates local state (the static viewer cannot write back to the database).

### Message Rendering

- Full message bodies (`body_md`) are loaded **on demand** per message click to keep initial load fast.
- Markdown is rendered to HTML with **Marked v11.0.0** and sanitized through **DOMPurify v3.0.8**.
- Prose styling (headings, code blocks, blockquotes, lists, tables, horizontal rules) is included for both light and dark modes with carefully chosen color tokens.
- Preview snippets for the message list are generated by stripping Markdown syntax to plain text and truncating.
- Search term highlighting is applied after HTML-entity escaping to prevent injection.

### Security

- **Content Security Policy** -- a `<meta>` CSP tag restricts script, style, font, image, and connect sources.
- **Trusted Types** -- two policies are registered: `mailViewerDOMPurify` (wraps DOMPurify for `createHTML` and restricts `createScriptURL` to `./vendor/`) and a `default` policy for Clusterize.js compatibility that passes through already-escaped HTML.
- **DOMPurify sanitization** on all Markdown-to-HTML rendering paths.
- **HTML entity escaping** (`escapeHtml`) applied to every user-controlled string (subjects, senders, recipients, snippets, timestamps) before DOM insertion via `innerHTML`.
- Vendor scripts pinned with **SHA-256 SRI hashes** in both `manifest.json` and `viewer/vendor_manifest.json`.

### Refresh Controls

- Manual refresh button and an optional **30-second auto-refresh** interval (simulated in static mode since the database is immutable). Interval is cleaned up on component destruction to prevent memory leaks.

### Hosting and Deployment

- `index.html` (root) -- meta-refresh + JavaScript redirect to `viewer/index.html`.
- `_headers` -- Cloudflare Pages / Netlify header rules for Cross-Origin Isolation (`Cross-Origin-Opener-Policy: same-origin`, `Cross-Origin-Embedder-Policy: require-corp`), plus correct MIME types for `.sqlite3` and chunk files.
- `coi-serviceworker.js` (v0.1.7) -- drop-in service worker for hosts like GitHub Pages that cannot set custom response headers; intercepts fetches and injects COOP/COEP headers client-side.
- `preview-reload.js` -- hot-reload helper for the `share preview` local development server. Polls a `/__preview__/status` endpoint every 2 seconds and triggers a page reload when the signature changes.
- `.nojekyll` -- disables Jekyll processing on GitHub Pages so `_headers` and `.wasm` files are served as-is.
- `HOW_TO_DEPLOY.md` -- deployment checklists covering GitHub Pages, Cloudflare Pages, Netlify, Amazon S3, and generic static hosts.
- `viewer/__init__.py` -- Python package marker so the MCP Agent Mail CLI can locate bundled viewer assets via `__path__`.

### Vendored Dependencies

| Library | Version | Purpose |
|---------|---------|---------|
| sql.js | 1.10.1 | SQLite compiled to WebAssembly |
| Marked | 11.0.0 | GitHub-Flavored Markdown parser |
| DOMPurify | 3.0.8 | HTML sanitizer for rendered Markdown |
| Clusterize.js | 0.18.0 | Virtual-list CSS retained; JS superseded by custom scroller |

### Frontend Stack

- **Alpine.js** -- reactive UI framework; the entire viewer is a single `viewerController()` Alpine component with `x-data`, `x-show`, `x-model`, `x-text`, `@click`, and `$watch` bindings. Alpine startup is deferred until the controller registers.
- **Tailwind CSS** (CDN) -- utility-first styling with a custom `primary` color scale and `fadeIn` / `slideIn` keyframe animations.
- **Lucide Icons** (CDN) -- icon set re-created on each virtual-scroll render pass.
- **Inter** (Google Fonts) -- primary UI typeface at weights 300--800.
