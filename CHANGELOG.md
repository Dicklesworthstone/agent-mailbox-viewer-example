## 2025-11-06 — Initial Mailbox Export

First commit. The full static bundle was generated by the MCP Agent Mail `share export` pipeline (exporter version `prototype`, schema `0.1.0`) on 2025-11-06.

([`9857721`](https://github.com/Dicklesworthstone/agent-mailbox-viewer-example/commit/98577212eb6c86780dc9ac5705ac88704d05f24a))

### Mailbox Snapshot

- `mailbox.sqlite3` — 4.7 MB scrubbed SQLite database containing 1,500 messages across 66 agents, exported from the `mcp_agent_mail_ios_app` project scope.
- Scrub preset `standard`: ack flags cleared (1,500), agent links removed (398), file reservations removed (289), recipients cleared (3,242). Agent names retained; no body redaction or pseudonymization.
- `manifest.json` — machine-readable export metadata: database SHA-256, scrub stats, project scope, hosting hints, viewer SRI hashes.
- `manifest.sig.json` — Ed25519 signature over the manifest for bundle integrity verification.

### Static Viewer (Alpine.js + Tailwind CSS)

The `viewer/` directory ships a fully self-contained single-page application (no build step, no server-side runtime).

#### Core capabilities

- **In-browser SQLite** via sql.js (v1.10.1) compiled to WebAssembly — the viewer loads `mailbox.sqlite3` directly in the browser and runs SQL queries client-side.
- **OPFS caching** — after first load, the database is written to the Origin Private File System so subsequent visits skip the network fetch. Cache state is SHA-256-validated and can be toggled on/off from the UI.
- **Chunked database support** — large databases can be split into numbered chunks that are fetched and reassembled transparently.

#### Viewer UI

- **Split / List / Threads view modes** with responsive mobile layout (media query breakpoint at 768 px, scroll-aware filter collapse, full-screen mobile message overlay).
- **Thread explorer** — groups messages by `thread_id`, shows message count per thread, supports thread search, and scrolls selected threads into view.
- **Message detail pane** — loads full `body_md` on demand, renders GitHub-Flavored Markdown (Marked v11.0.0) with DOMPurify (v3.0.8) sanitization and Trusted Types integration.
- **Virtual scrolling** — custom implementation with estimated-row-height refinement, overscan, and `requestAnimationFrame`-throttled rendering to handle large inboxes without DOM bloat (replaces earlier Clusterize.js approach).
- **Advanced search** — supports parentheses, `AND`/`OR`/`NOT` operators, quoted phrases, with shunting-yard parser producing an AST. Uses FTS5 `MATCH` when available, falls back to `LIKE` queries. Debounced at 140 ms.
- **Multi-axis filtering** — filter by project, sender, recipient, importance level (urgent/high/normal/low), thread membership, and message kind (user vs. administrative). Administrative messages are auto-detected by subject/body pattern matching.
- **Sorting** — newest, oldest, subject (alpha), sender (alpha), longest body.
- **Bulk actions** — select all / individual message checkbox selection, mark-as-read in local state.
- **Dark mode** — toggle with `localStorage` persistence; respects `prefers-color-scheme` on first visit; FOUC-free via inline `<script>` in `<head>`.
- **Importance badges** — color-coded pills for urgent (red) and high (yellow) priority messages.
- **Project badges** — deterministic hash-based color assignment for consistent per-project visual identity.
- **Refresh controls** — manual refresh button and optional 30-second auto-refresh interval (simulated in static mode).
- **Query plan diagnostics** — `explainMode` flag logs `EXPLAIN QUERY PLAN` output for every SQL query to the browser console.

#### Security

- Content Security Policy meta tag restricting script/style/font/image sources.
- Trusted Types policies (`mailViewerDOMPurify` + default) for `innerHTML` safety.
- DOMPurify sanitization on all Markdown-to-HTML paths.
- HTML entity escaping (`escapeHtml`) on every user-controlled string before DOM insertion.
- Vendor scripts pinned with SHA-256 SRI hashes in `manifest.json` and `viewer/vendor_manifest.json`.

#### Vendored Dependencies

| Library | Version | Purpose |
|---------|---------|---------|
| sql.js | 1.10.1 | SQLite compiled to Wasm |
| Marked | 11.0.0 | Markdown parser |
| DOMPurify | 3.0.8 | HTML sanitizer |
| Clusterize.js | 0.18.0 | Virtual list (CSS retained; JS superseded by custom implementation) |

### Hosting and Deployment

- `index.html` (root) — meta-refresh + JS redirect to `viewer/index.html`.
- `_headers` — Cloudflare Pages / Netlify header rules for Cross-Origin Isolation (`COOP: same-origin`, `COEP: require-corp`), proper MIME types for `.sqlite3` and chunk files.
- `coi-serviceworker.js` — drop-in service worker for GitHub Pages (which cannot set response headers); adds COOP/COEP headers client-side.
- `preview-reload.js` — hot-reload helper for the `share preview` local development server.
- `.nojekyll` — disables Jekyll processing on GitHub Pages so `_headers` and `.wasm` files are served as-is.
- `HOW_TO_DEPLOY.md` — deployment checklists for GitHub Pages, Cloudflare Pages, Netlify, Amazon S3, and generic static hosts.
- `viewer/__init__.py` — Python package marker so the MCP Agent Mail CLI can locate bundled viewer assets.
